# Internationalization (i18n)

@llmnative/react includes a built-in i18n layer. Every user-visible string in the framework is defined in a typed dictionary and resolved via `useI18n()`. Locale switching is persisted in a cookie and works without a page reload.

The default locale is `en`. Pass `locale` and optional `translations` to `<App>` to switch or extend.

---

## Quick start

```tsx
<App
  locale="it"
  translations={{
    it: {
      common: { search: 'Cerca' },
      form: { buttonSave: 'Salva', buttonDelete: 'Elimina' },
    },
  }}
/>
```

Strings not overridden fall back to English. Add as many locales as you need, each as a partial dictionary.

---

## Configuration

The `locale` prop and `translations` map are the primary API for app-level configuration.

### Switching locale at config time

```tsx
<App locale="de" />
```

`locale` selects the active locale on first render. If not set, the framework checks a persisted cookie — if absent, defaults to `en`.

### App-level translations

```tsx
<App
  translations={{
    it: { /* partial I18nDict */ },
    de: { /* partial I18nDict */ },
  }}
/>
```

Translations are deep-merged on top of the built-in English dictionary. Any key you omit stays in English.

### Switching locale at runtime

Use `<LocaleSwitcher>` to let the user change locale. The switch is immediate and persisted in a cookie (`llmnative_locale`, 1 year).

```tsx
import { LocaleSwitcher } from '@llmnative/react';

<LocaleSwitcher
  labels={{
    en: 'EN',
    it: 'IT',
    de: 'DE',
  }}
/>
```

The `LocaleSwitcher` uses `useI18n().setLocale()` internally.

## The `useI18n()` hook

Two overloads — call without arguments for the full controller, or pass a dot-notation namespace to get a typed slice.

### Full controller

```tsx
import { useI18n } from '@llmnative/react';

const { dict, locale, setLocale, availableLocales } = useI18n();

console.log(dict.form.buttonSave); // "Save"
```

### Namespace slice (typed)

```tsx
const formDict = useI18n('form');
//    ^? I18nDict['form'] — all form strings, autocompleted

const buttonSave = useI18n('form.buttonSave');
//    ^? string
```

Using the namespace overload is preferred in component code — it reduces re-render scope and keeps the dependency explicit.

## Dictionary structure

The full `I18nDict` type is defined in `src/I18n.tsx` and covers every user-visible string in the framework.

```ts
interface I18nDict {
  common:        { save, cancel, delete, close, back, search, loading, noDataFound, … };
  auth:          { signIn, signOut, connect, connected, authenticated, … };
  form:          { headerAdd, headerEdit, buttonSave, buttonDelete, requiredField, … };
  grid:          { buttonAdd, deleteConfirm, emptyState };
  select:        { placeholder };
  modal:         { save, delete, cancel, close };
  upload:        { clickOrDrag, dropToUpload, uploadMore, editFileName, … };
  notifications: { title, seeAll };
  code:          { copyCode, copy, copied, codeLanguageDefault };
  table:         { noDataFound, selectAllRows, sortBy, sortByCurrent, selectRow, … };
  gallery:       { selectItem };
  crop:          { enableCrop, variants, outputFile, active, removeVariant, fileName };
  imageEditor:   { title, undo, redo, zoomOut, zoomIn, crop, flipHorizontal, … };
  prompt:        { noProviders, aiNotConfiguredEdit, run, tokenUsage, … };
  layout:        { maxElements, noSpace, dragToMove, remove, dragToResize, dragHere };
}
```

Each locale file must match this shape. Use `defineLocaleMessages` for type-safe creation:

```ts
import { defineLocaleMessages } from '@llmnative/react';

export const fr = defineLocaleMessages({
  common: { save: 'Enregistrer' },
  // TypeScript ensures only valid keys
});
```

## Runtime translation registration

Translations can be added or overridden after the app has mounted — useful for lazy-loaded modules, plugin systems, or user-provided dictionaries.

```tsx
import { useI18n } from '@llmnative/react';

const { registerTranslations } = useI18n();

// Load and register a locale file dynamically
const frDict = await import('./locales/fr');
registerTranslations('fr', frDict);
```

`registerTranslations` deep-merges with any existing translations for that locale, so multiple modules can contribute independently.

## Interpolation

Strings can contain `{variable}` placeholders. Use the `interpolate` helper to substitute values — it works independently of the hook:

```tsx
import { interpolate } from '@llmnative/react';

interpolate('Sort by {label}', { label: 'Name' });
// → "Sort by Name"
```

When a variable is missing, the placeholder is kept verbatim so you can spot the gap during development.

---

## Adding i18n to a component

Components should use `useI18n('namespace')` instead of hardcoding user-visible strings.

```tsx
const MyBadge = ({ type }) => {
  const dict = useI18n('common');
  return <span>{dict.loading}</span>;
};
```

If the namespace does not exist yet, extend `I18nDict` in `src/I18n.tsx` and add the default `en` entry in `src/conf/i18n/en.ts`.

| `it` | `src/conf/i18n/it.ts` |
| `de` | `src/conf/i18n/de.ts` |
| `ru` | `src/conf/i18n/ru.ts` |
| `zh` | `src/conf/i18n/zh.ts` |
| `ar` | `src/conf/i18n/ar.ts` |

Import them for use as a base or reference:

```ts
import { en, it, de } from '@llmnative/react/locales';
```

## Cookie persistence

Locale is persisted in a cookie named `llmnative_locale` with a 1-year max-age. On subsequent visits the cookie is read before falling back to the `locale` prop, then to `'en'`. This means a user who switches locale returns to their choice even if the app does not pass a specific `locale` prop.
