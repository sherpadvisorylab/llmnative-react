# Theme system

The theme system has two independent layers that you can configure separately or together:

| Layer | What it controls | Configured via |
|-------|-----------------|----------------|
| **CSS tokens** | Colors, border radius, font family | `ThemePresetConfig` |
| **Component classes** | Tailwind class overrides per component | `ThemeConfig` |

Both layers are configured via the `themeProvider` prop on `<App>` and can be changed at runtime via `useThemeController()`. The CSS token layer is applied immediately at mount via CSS custom properties on `:root`; the component class layer is merged into component internals at render time.

---

## Built-in presets

Three presets ship out of the box. Each defines a complete light and dark palette so switching color mode never loses the preset's hue identity. Pass the preset name as a string shorthand — no additional configuration needed.

```tsx
<App themeProvider="default" />   // clean, rounded, blue primary (default)
<App themeProvider="flat" />      // sharp corners, slate tones
<App themeProvider="cyber" />     // zero radius, green neon, dark-first
```

| Preset | Radius | Primary | Character |
|--------|--------|---------|-----------|
| `default` | 0.5 rem | Blue | shadcn-style, neutral |
| `flat` | 0.125 rem | Slate | administrative, dense |
| `cyber` | 0 rem | Green | neon, hacker aesthetic |

## Scaffolded project

When you run `npx react-firestrap create` and pick a theme, the scaffold writes the preset name into `src/conf/app.ts` and passes it as a string to `<App>`. No extra file is generated — the built-in preset is used directly.

```ts
// src/conf/app.ts  — generated by the scaffold
export const appConfig = {
  theme: import.meta.env.VITE_THEME ?? 'flat',
  // …
};
```

```tsx
// src/index.tsx
<App themeProvider={appConfig.theme} />
```

To customize colors or component classes beyond the preset, replace the string with an object or an async import:

```tsx
// switch to a full custom config
<App themeProvider={{ defaultPreset: 'flat', theme: { Form: { buttonSaveClass: 'btn-primary' } } }} />

// or load from a separate file (keeps first render synchronous)
<App themeProvider={() => import('./conf/theme')} />
```

See [Scaffold output](/docs/scaffolding) for the full generated structure.

## Runtime control

`useThemeController()` gives full programmatic access to every theme axis — color mode, palette, radius and font — without a page reload. Use it in settings panels, keyboard shortcuts or live customization tools. The hook must be called inside `<App>`.

Each setter is independent: `setRadius` only changes border radius, `applyPreset` only changes colors. Radius and font are never reset by a preset switch.

```tsx
import { useThemeController } from 'react-firestrap';

function ThemeSettings() {
  const {
    resolvedMode,   // 'light' | 'dark' — actual mode after 'system' resolution
    preset,         // id of the active preset, e.g. 'default'
    primary,        // active primary color as HSL string, e.g. '221.2 83.2% 53.3%'
    radius,         // active border radius in rem
    fontSans,       // active sans-serif font stack
    fontMono,       // active monospace font stack
    toggleMode,     // () => void — toggles between light and dark
    applyPreset,    // (id: string) => void — changes colors only, preserves radius & font
    setPrimary,     // (hsl: string) => void — also derives secondary, accent and ring
    setRadius,      // (rem: number) => void
    setFont,        // (sans: string, mono?: string) => void
    setTokens,      // (Partial<ColorScale>) => void — bulk token update
  } = useThemeController();

  return (
    <>
      <button onClick={toggleMode}>
        {resolvedMode === 'light' ? 'Dark' : 'Light'}
      </button>
      <button onClick={() => applyPreset('flat')}>Flat</button>
      <button onClick={() => setPrimary('262.1 83.3% 57.8%')}>Violet</button>
      <button onClick={() => setRadius(0)}>Sharp</button>
      <button onClick={() => setFont("'Inter', sans-serif")}>Inter</button>
    </>
  );
}
```

> `applyPreset` changes only the color palette. Radius and font are independent axes — they persist unchanged when you switch presets.

> `setPrimary` automatically derives `secondary`, `accent` and `ring` from the same hue so the palette stays coherent when you pick a custom color.

## Custom preset

Register additional named presets alongside the built-ins. Use this when your app has a brand palette that should live alongside `default` / `flat` / `cyber` and be switchable at runtime via `applyPreset`. All `colors` and `dark` fields are optional — omitted tokens fall back to `default` preset values.

```tsx
<App
  themeProvider={{
    defaultMode: 'light',
    defaultPreset: 'brand',
    presets: {
      brand: {
        radius: 0.75,
        fontSans: "'Plus Jakarta Sans', sans-serif",
        colors: {
          primary:             '346.8 77.2% 49.8%',
          primaryForeground:   '0 0% 100%',
          secondary:           '346.8 40% 96%',
          secondaryForeground: '346.8 47% 11%',
          background:          '0 0% 100%',
          foreground:          '240 10% 3.9%',
          // … any other ColorScale token
        },
        dark: {
          primary:    '346.8 77.2% 60%',
          background: '240 10% 3.9%',
          // … dark-mode overrides — merged on top of colors
        },
        theme: {
          Card: { headerClass: 'bg-primary/5 font-semibold' },
          Form: { buttonSaveClass: 'btn-primary' },
          Menu: { linkClass: 'nav-link px-3 rounded-md' },
        },
      },
    },
  }}
/>
```

## `ThemePresetConfig` — full interface

The configuration object for a single preset. Pass it directly as `themeProvider` or nest it inside `presets` when registering multiple named presets. All fields are optional — omit any field to inherit the `default` preset value. The `theme` field scopes `ThemeConfig` overrides to this preset only, so component classes can vary per preset.

```ts
interface ThemePresetConfig {
  /** Initial color mode. 'system' follows the OS preference. Default: 'light' */
  mode?: 'light' | 'dark' | 'system';

  /** Base border-radius in rem. Scales all rounded-* utilities proportionally. Default: 0.5 */
  radius?: number;

  /** CSS font-family for body text. Injected as --font-sans custom property. */
  fontSans?: string;

  /** CSS font-family for code/mono elements. Injected as --font-mono custom property. */
  fontMono?: string;

  /** Full semantic color palette for light mode. Values are HSL channel strings "H S% L%". */
  colors?: ColorScale;

  /** Color overrides applied in dark mode — merged on top of colors. Only supply what differs. */
  dark?: ColorScale;

  /** Arbitrary CSS custom properties beyond ColorScale, e.g. spacing or branding tokens. */
  variables?: Record<string, string>;

  /** Component class overrides scoped to this preset. Merged with any global ThemeConfig. */
  theme?: ThemeConfig;
}
```

## `ColorScale` — all 25 tokens

Every value is an HSL channel string **without** the `hsl()` wrapper, e.g. `"221.2 83.2% 53.3%"`. At runtime each token is injected as a `--rf-{token}` CSS custom property: on `:root` for light mode and on `.dark` for dark mode. All values are optional — omit any token to keep the preset default.

```ts
interface ColorScale {
  // ── Surfaces ──────────────────────────────────────────────────
  background?:          string;  // page background
  foreground?:          string;  // default text color

  card?:                string;  // card / panel surface
  cardForeground?:      string;

  popover?:             string;  // dropdown / tooltip surface
  popoverForeground?:   string;

  // ── Brand ─────────────────────────────────────────────────────
  primary?:             string;  // main action color (buttons, links, focus)
  primaryForeground?:   string;  // text on primary backgrounds

  secondary?:           string;  // secondary action / surface tint
  secondaryForeground?: string;

  // ── Neutral ───────────────────────────────────────────────────
  muted?:               string;  // subtle background (inputs, disabled states, tabs)
  mutedForeground?:     string;  // subdued / placeholder text

  accent?:              string;  // hover highlight / selected state background
  accentForeground?:    string;

  // ── Status ────────────────────────────────────────────────────
  destructive?:         string;  // danger / delete actions
  destructiveForeground?: string;

  success?:             string;
  successForeground?:   string;

  warning?:             string;
  warningForeground?:   string;

  info?:                string;
  infoForeground?:      string;

  // ── Form & focus ──────────────────────────────────────────────
  border?:              string;  // default border color
  input?:               string;  // input field border (often slightly different from border)
  ring?:                string;  // keyboard focus ring
}
```

## `ThemeConfig` — full component overrides

Override the Tailwind class strings that react-firestrap components render internally. Every field is optional and deeply merged with preset defaults — you only specify what you want to change. This is the right place for things like custom button variants, adjusted menu spacing or a branded card header style.

For most components the keys follow a consistent pattern: `wrapClass` targets the outermost container, `className` the main element, and `*Class` suffixes target named inner slots (header, body, footer, etc.).

```ts
interface ThemeConfig {
  // ── Data components ───────────────────────────────────────────
  Grid?: {
    Card?:    CardTheme;
    Table?:   TableTheme;
    Gallery?: GalleryTheme;
    Modal?:   ModalTheme & { mode?: 'form' | 'empty' };
    i18n?: {
      headerAdd?:  string;  // default: 'Aggiungi'
      headerEdit?: string;  // default: 'Modifica'
      buttonAdd?:  string;  // default: 'Aggiungi'
    };
  };
  Table?:   TableTheme;
  Gallery?: GalleryTheme;

  Pagination?: {
    wrapClass?:      string;
    className?:      string;
    stickyClass?:    string;
    scrollToTop?:    boolean;
    scrollBehavior?: string;
    maxItems?:       number;
    sticky?:         boolean;
    align?:          string;
  };

  Carousel?: {
    showIndicators?: boolean;
    showControls?:   boolean;
    showCaption?:    boolean;
    layoutDark?:     boolean;
    autoPlay?:       boolean;
  };

  // ── UI primitives ─────────────────────────────────────────────
  Card?:   CardTheme;
  Modal?:  ModalTheme;

  Alert?:  { className?: string };
  Badge?:  { className?: string };

  Loader?: {
    wrapClass?:   string;
    className?:   string;
    icon?:        string;   // icon name passed to <Icon>
    title?:       string;
    description?: string;
  };

  ActionButton?: {
    className?:  string;
    badgeClass?: string;
  };
  LoadingButton?: {
    className?:   string;
    badgeClass?:  string;
    spinnerClass?: string;
  };
  LinkButton?: { className?: string };

  // ── Navigation ────────────────────────────────────────────────
  Menu?: MenuTheme;

  Dropdown?: {
    wrapClass?:        string;
    className?:        string;
    buttonClass?:      string;
    badgeClass?:       string;
    menuClass?:        string;
    menuHeaderClass?:  string;
    menuItemClass?:    string;
    menuDividerClass?: string;
    headerClass?:      string;
    footerClass?:      string;
    Menu?:             MenuTheme;  // overrides for the nested Menu inside Dropdown
  };

  Notifications?: {
    wrapClass?: string;
    Dropdown?: {
      className?:   string;
      buttonClass?: string;
      menuClass?:   string;
    };
  };

  Brand?: {
    wrapClass?:  string;
    className?:  string;
    logoClass?:  string;
    labelClass?: string;
  };

  SignIn?: {
    className?:   string;
    avatarClass?: string;
  };

  // ── Form ──────────────────────────────────────────────────────
  Form?: {
    wrapClass?:         string;
    buttonSaveClass?:   string;   // e.g. 'btn-primary' or 'btn-outline-primary border-0'
    buttonDeleteClass?: string;
    buttonBackClass?:   string;
    Card?: {
      headerClass?: string;
      bodyClass?:   string;
      footerClass?: string;
    };
    i18n?: {
      headerNewRecord?:      string;
      buttonSave?:           string;
      buttonDelete?:         string;
      buttonBack?:           string;
      noticeRequiredFields?: string;
    };
  };

  Select?:       { wrapClass?: string; className?: string };
  Autocomplete?: { wrapClass?: string; className?: string };

  // ── Media & misc ──────────────────────────────────────────────
  Image?:       { wrapClass?: string; className?: string };
  ImageAvatar?: { wrapClass?: string; className?: string };
  Percentage?:  { wrapClass?: string; className?: string };
  Tab?:         { wrapClass?: string; className?: string };
  Code?:        { wrapClass?: string; className?: string };
  Prompt?:      { wrapClass?: string; className?: string };
}

// ── Sub-interfaces ────────────────────────────────────────────────

interface MenuTheme {
  wrapClass?:    string;
  className?:    string;
  headerClass?:  string;
  itemClass?:    string;
  linkClass?:    string;   // the <a> or <button> inside each menu item
  iconClass?:    string;
  textClass?:    string;
  badgeClass?:   string;
  arrowClass?:   string;   // expand/collapse arrow for submenus
  submenuClass?: string;
}

interface CardTheme {
  wrapClass?:   string;
  className?:   string;
  headerClass?: string;
  bodyClass?:   string;
  footerClass?: string;
  showLoader?:  boolean;  // whether to show a loading overlay while data is fetched
  showArrow?:   boolean;  // whether to show a chevron in the header
}

interface TableTheme {
  wrapClass?:     string;
  className?:     string;  // the <table> element
  headerClass?:   string;
  bodyClass?:     string;
  footerClass?:   string;
  scrollClass?:   string;  // wraps the table for horizontal scroll
  selectedClass?: string;  // applied to the selected row <tr>
}

interface GalleryTheme {
  wrapClass?:     string;
  className?:     string;
  scrollClass?:   string;
  headerClass?:   string;
  bodyClass?:     string;
  footerClass?:   string;
  selectedClass?: string;
  gutterSize?:    string;  // Bootstrap gutter class, e.g. 'g-3'
  rowCols?:       string;  // Bootstrap row-cols class, e.g. 'row-cols-3'
}

interface ModalTheme {
  size?:          string;   // e.g. 'sm' | 'lg' | 'xl' | 'fullscreen'
  position?:      string;   // e.g. 'center' | 'top'
  wrapClass?:     string;
  className?:     string;   // the .modal-dialog element
  headerClass?:   string;
  titleClass?:    string;
  subTitleClass?: string;
  bodyClass?:     string;
  footerClass?:   string;
  iconExpand?:    string;   // icon name for the expand button
  iconCollapse?:  string;   // icon name for the collapse button
}
```
